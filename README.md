# L476-DAC3-TriangularWaveform
# 📘 STM32 DAC Triangle Wave Generator (Timer Triggered)

## 📌 Overview
This project demonstrates how to generate a **triangle waveform** using the internal DAC of an STM32 microcontroller. The waveform is triggered by **TIM2 (Timer 2)** using TRGO (Trigger Output), allowing fully hardware-driven signal generation.

---

## ⚙️ Hardware Requirements
- STM32 Nucleo Board (e.g., NUCLEO-L476RG)
- Oscilloscope or Arduino (for waveform visualization)
- Jumper wires

---

## 🧠 Key Concepts
- DAC (Digital-to-Analog Converter)
- Timer Trigger (TIM → DAC)
- Triangle Wave Generation (Hardware Mode)
- TRGO (Trigger Output)

---

## 🔧 Configuration

### DAC
- Channel: DAC_CHANNEL_1
- Trigger: TIM2 TRGO
- Output Buffer: Enabled
- Wave Generation: Triangle
- Amplitude: 2047 (12-bit)

```c
HAL_DACEx_TriangleWaveGenerate(&hdac1, DAC_CHANNEL_1, DAC_TRIANGLEAMPLITUDE_2047);

TIM2
Prescaler: 390
Period: 1
Counter Mode: Down
TRGO: Update Event

sMasterConfig.MasterOutputTrigger = TIM_TRGO_UPDATE;

🚀 How It Works
TIM2 runs continuously
Each update event generates a TRGO signal
DAC receives trigger and updates output
Hardware generates triangle waveform automatically
Output is available on PA4 (DAC OUT1)

Frequency Formula
f_update = f_timer / ((PSC + 1) × (ARR + 1))

Example:

Clock ≈ 80 MHz
PSC = 390
ARR = 1
f ≈ 80,000,000 / (391 × 2) ≈ 102 kHz

🧪 Testing
Arduino
PA4 → A0
Use Serial Plotter to visualize signal

Main Code Snippet
uint32_t Offset = 1247;

HAL_TIM_Base_Start(&htim2);
HAL_DAC_Start(&hdac1, DAC_CHANNEL_1);
HAL_DAC_SetValue(&hdac1, DAC_CHANNEL_1, DAC_ALIGN_12B_R, Offset);
⚠️ Notes
PA4 must be configured as Analog mode
DAC output depends on reference voltage (typically 3.3V)
Waveform is generated by hardware, not CPU
Timer controls waveform speed
